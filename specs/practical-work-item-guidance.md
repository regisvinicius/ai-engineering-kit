# Practical Work Item Guidance

## Goal

Help users and AI agents understand the day-to-day flow for executing a new user story or bug, and provide a concrete Node example they can follow.

## Context

The repository already includes generic workflows and task/spec scaffolding, but it does not yet provide one practical checklist that ties those pieces together. It also benefits from a realistic example of how a new feature should be documented in a generated project.

## Requirements

- Add a practical workflow that explains how to execute a new user story or bug from start to finish
- Include a concrete example for a Node project that implements a basic User CRUD
- Make the example available in generated projects
- Surface the new workflow and example in the main documentation and generated project README

## Acceptance Criteria

- Generated projects include a workflow for executing a work item end to end
- Generated projects include a practical Node User CRUD task/spec example
- Root and generated documentation point users to the new workflow and example
