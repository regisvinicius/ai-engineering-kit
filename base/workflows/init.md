# Initialization Workflow

## Purpose

Gather context about the repository before starting any implementation.

Agents should understand the codebase, architecture, and conventions before making changes.

---

## Triggers

Use this workflow when:

- starting work on a new task
- exploring an unfamiliar codebase
- implementing a new feature
- investigating an issue

---

## Workflow

### 1. Repository Exploration

- Explore the project structure
- Identify main modules and directories
- Locate entry points (server, app, main)
- Understand how the project is organized

Focus on:

- `/src`
- `/docs`
- `/specs`
- `/tasks`

---

### 2. Architecture Understanding

- Identify architectural patterns used
- Understand separation of concerns
- Locate controllers, services, repositories, or equivalents
- Identify configuration files

---

### 3. Code Pattern Analysis

- Find similar implementations
- Identify reusable components
- Understand naming conventions
- Observe error handling patterns

---

### 4. Dependency Awareness

- Identify external dependencies
- Understand build or runtime requirements
- Check configuration files (package.json, pyproject.toml, etc.)

---

### 5. Context Documentation

Summarize key findings:

- important modules
- relevant file paths
- architectural patterns
- constraints or assumptions

---

## Output

Provide:

- context summary
- relevant file locations
- architectural observations
- potential risks or constraints
